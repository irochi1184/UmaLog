# UmaLog
あなたの「負け方」を静かに見える化する競馬ログアプリ

## アプリ概要：うまログ
うまログは、競馬での「負け方」を振り返り、自分がどんな買い方で負けやすいのかを可視化するiOSアプリです。予想や的中を狙うのではなく、過去の馬券購入履歴をもとに感情的・無意識に繰り返している負けパターンに気づくことを目的としています。

## コンセプト
- 当てるための競馬アプリではない
- 勝てない理由を冷静に知るためのアプリ
- 競馬を“やめさせる”のではなく、“続け方を整える”アプリ
- 「記録・振り返り・分析」に特化し、外部データや予想ロジックには依存しない

## 想定ユーザー
- 競馬を継続的に楽しんでいる人
- なんとなく負けている自覚はあるが、理由が分からない人
- 収支や回収率を感覚ではなく数字で把握したい人
- 競馬ガチ勢〜ライト層まで（全年代対応）

## 主な機能（MVP想定）
### 馬券記録
- 1レースあたり30秒程度で入力できるシンプルな記録
- 馬券種（単勝 / 複勝 / 馬連 / 三連系など）
- 人気帯（1〜3番人気 / 中穴 / 穴）
- レース格（G1 / 重賞 / 平場）
- 時間帯（午前 / 午後）
- 投資額 / 払戻額
- 馬名・オッズ・競馬場の入力は不要

### 振り返り・分析
- 記録されたデータをもとに回収率を自動算出
- 条件別（馬券種 × 人気帯 × 時間帯など）の成績を算出
- 最も負けているパターンを抽出し、「午後 × 穴狙い で最も負けています」のように短く断定的に表示

### マネタイズ方針（構想）
- 基本の記録・当月収支は無料
- 条件別分析・過去月分析・詳細な傾向表示は有料（サブスク）
- 「分析＝価値」として、競馬を続けるユーザーほど解約しづらい設計を想定

## デザイン・世界観
- アプリ名：うまログ
- アイコン：芝生を連想させるグリーン背景に白の馬シルエット。ロゴ記号は入れずシンプルで知的な印象。
- SwiftUI + SwiftData（ローカル保存）で最小限の画面構成
  - ホーム（今月の回収率＋一言分析）
  - 記録追加
  - 分析（有料構想）
  - 設定 / 課金管理

## 使い方（開発中の最小プロトタイプ）
1. フォームから「馬券種・人気帯・レース格・時間帯・投資額・払戻額」を入力し「記録を追加」。
2. 画面上部で回収率と記録件数を確認。
3. 記録が溜まると「午後 × ○○」など負けやすいパターンを一言で表示。

> 本アプリは「勝ち方」ではなく「負け方」に焦点を当てています。感情的な買い方を手放し、続け方を整えるための静かな鏡としてご活用ください。
